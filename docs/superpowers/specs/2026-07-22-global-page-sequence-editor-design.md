# 全局页面编排交互重构设计

日期：2026-07-22

## 背景

当前“全局配置 → 页面列表”使用横向换行的碎片化控件：页面按钮、上移/下移、复制、删除和相邻页对比按钮散落在同一流式容器中。页面增加后，顺序、操作归属和对比关系不易扫描；对比页虽然在数据上成对存在，但界面上没有形成稳定的组合心智。

本次重构将页面列表改成单列编排台：一行一个页面 bar，直接拖动排序；开启对比的两个页面呈现为不可拆分的原子组。同时重新组织背景音乐、总时长、逐页明细和全局预览，形成连续的“编排 → 配乐 → 核对 → 预览”工作流。

## 目标

- 一行只展示一个页面 bar，页面顺序可快速扫描。
- 使用拖拽替代上移/下移按钮，并支持鼠标、触控和键盘操作。
- 对比页具有清楚的链条绑定提示，拖动任一成员时整组移动。
- 复制、删除和“与下一页对比”统一位于 bar 最右侧。
- 排序后同步更新页面顺序、对比索引和当前选中页。
- 压缩低频信息，避免背景音乐和逐页时长明细抢占页面编排空间。
- 不改变现有持久化配置格式。

## 非目标

- 不允许跨位置保持对比关系；对比仍只发生在相邻页面之间。
- 不允许在绑定组内部交换两个页面的顺序。
- 不重构对比动画、视频渲染或全局覆盖配置本身。
- 不引入页面持久化 ID，也不迁移历史配置。

## 方案选择

采用“单列编排台”方案。

未采用双栏控制台，因为当前右侧编辑面板宽度有限，双栏会压缩页面名称和右侧操作区，并增加响应式回退逻辑。未采用独立卡片分区，因为重复的边框与标题会拉长页面，削弱从编排到预览的连续感。

## 信息架构

“全局配置”卡片内部按以下顺序排列：

1. 标题区：标题、简短拖拽提示、“添加页面”按钮。
2. 页面编排：单列 bar、对比绑定组和拖拽落点。
3. 背景音乐：紧凑资源行，保留选择、替换和上传能力。
4. 时长摘要：总秒数、总帧数、页面数和“时长明细”折叠入口。
5. 全局预览：横向占满的主要结束动作。

逐页时长明细默认折叠。展开后继续沿用现有的单页/对比组时长计算结果，但以紧凑列表呈现。

## 页面 bar

每个 bar 从左到右包含：

- 拖拽柄；
- 两位序号；
- 页面名称；
- 右对齐操作区：复制、删除、与下一页对比。

点击 bar 的非操作区域切换当前页面。只有拖拽柄启动排序，避免点击页面或操作按钮时误拖。当前页使用现有主色高亮；普通页使用中性背景。

空间不足时，操作区先隐藏图标旁的文字，只保留图标、tooltip 和可访问名称。bar 本身保持单行，不因响应式收窄变成多行卡片。

## 对比绑定组

开启对比的两个相邻页面组成一个不可拆分的原子组。组内保留两条独立 bar，并使用以下三重提示表达关系：

- 暖色外框与左侧强调线；
- 两条 bar 之间的链条桥接图形；
- “对比绑定 · 整组拖动”文字。

拖动任一成员的拖拽柄时，拖拽预览显示完整两页组。有效落点只出现在普通页或其他绑定组之前/之后，不出现在组内部。组内页面顺序固定；若需交换第一角色与第二角色，用户必须先解除对比、拖动换位，再重新建立对比。

## 操作规则

| 操作 | 普通页 | 对比组首项 | 对比组次项 |
| --- | --- | --- | --- |
| 复制 | 在当前页后插入未绑定副本 | 在整个绑定组后插入首项的未绑定副本 | 在整个绑定组后插入次项的未绑定副本 |
| 删除 | 删除当前页；单页时禁用 | 先确认；删除首项、解除对比、保留次项 | 先确认；删除次项、解除对比、保留首项 |
| 对比 | 与下一张未绑定普通页建立对比 | 显示“解除对比” | 禁用并提示“已参与对比” |

末页没有下一页时，对比按钮禁用并解释原因。普通页的下一项若为绑定组，也不能建立新对比。删除确认文案必须明确指出对比将被解除，但只删除被点选的页面。

## 背景音乐、时长和预览

背景音乐从大面积空状态改为紧凑资源行：左侧显示音乐图标、标题和当前文件/空状态，右侧显示选择或替换操作。现有刷新、上传和播放能力继续保留，可通过紧凑变体或次级操作呈现，不改变资源选择的数据契约。

时长区域改为单行摘要，展示：

- 总时长（秒）；
- 总帧数；
- 页面数；
- 时长明细折叠按钮。

全局预览按钮紧跟摘要，显示“全局预览 · N 页拼接”。页面顺序或对比关系改变后，摘要、明细与预览输入都从同一份最新配置重新计算，不维护第二份派生状态。

## 组件边界

### `GlobalConfigEditor`

保留为全局配置工作区的编排容器，负责组织页面编排、背景音乐、时长摘要、全局预览和现有的全局覆盖面板。它不再直接实现每条 bar 的拖拽细节。

### `PageSequenceEditor`

新增专用组件，负责：

- 普通页与对比组的视觉呈现；
- 当前页选择；
- 拖拽传感器、浮层和落点；
- 复制、删除、建立/解除对比按钮；
- 删除确认交互；
- 禁用态、tooltip 和可访问文案。

拖拽实现采用支持 pointer、touch 和 keyboard sensor 的 sortable 方案。实施阶段优先使用 `@dnd-kit/core` 与 `@dnd-kit/sortable`，并在落地前通过当前官方文档确认 React 19 下的推荐 API。

### 页面序列纯函数模块

新增纯函数模块，将 `pages + comparisons` 派生为页面序列项：

```ts
type PageSequenceItem =
  | { type: "page"; pageIndex: number }
  | { type: "comparison"; firstPageIndex: number; secondPageIndex: number };
```

该模块统一负责：

- 构建普通页/对比组序列；
- 按序列项整体重排；
- 生成旧页面索引到新页面索引的映射；
- 重建 `pages` 和 `comparisons`；
- 映射 `activePageIndex`；
- 计算复制页的插入边界；
- 删除绑定成员时解除相应对比。

UI 组件不自行拼接索引，避免排序、复制和删除各自实现一套不一致的重映射逻辑。

## 数据流

1. `GlobalConfigEditor` 从 `config.pages` 和 `config.comparisons` 派生 `PageSequenceItem[]`。
2. `PageSequenceEditor` 以序列项为 sortable 单位显示页面或对比组。
3. 拖拽结束时，将源序列项位置和目标边界交给纯函数模块。
4. 纯函数模块一次性返回新页面数组、新对比数组和新的当前页索引。
5. `RadarEditor` 在一次状态更新中提交新配置并同步当前页。
6. 时长摘要、预览和自动保存从更新后的配置自然重算。

拖拽过程中不修改持久化配置；只维护临时的活动项与目标落点。取消拖拽、落点无效或原位置未变化时不触发配置更新。

## 无障碍与动效

- 拖拽柄可聚焦，并提供明确的 `aria-label`。
- 键盘使用空格拿起、方向键移动、空格放下、Esc 取消。
- 屏幕阅读器播报拿起的页面/绑定组、当前位置和放置结果。
- 禁用按钮必须通过 tooltip 或描述文本解释原因。
- 绑定关系不能只依赖颜色表达。
- 拖动时使用轻微浮起、阴影和落点线；`prefers-reduced-motion` 下移除非必要过渡。

## 防御性行为

- 非法或过期的拖拽目标视为取消，不修改配置。
- 排序函数验证源项、目标边界和对比组完整性。
- 页面只剩一页时继续禁止删除。
- 用户取消删除确认时，不解除对比也不删除页面。
- 本次重构不自动修复历史配置中的非相邻或重叠对比；现有配置校验职责保持不变。

## 测试设计

### 单元测试

纯函数模块覆盖：

- 普通页跨过普通页；
- 普通页跨过绑定组；
- 绑定组跨过普通页；
- 两个绑定组互换；
- 绑定组内部不存在落点；
- 页面、对比索引和当前页索引同步映射；
- 复制任一绑定成员时副本位于组后且未绑定；
- 删除任一绑定成员时只删除目标页并解除对比；
- 取消或无效移动保持引用数据不变。

`PageSequenceEditor` 组件测试覆盖：

- 普通、当前和绑定状态的渲染；
- 点击 bar 选择页面；
- 右侧操作回调；
- 建立、解除和禁用对比；
- 删除确认与取消；
- 拖拽取消和键盘排序入口；
- 可访问名称与禁用原因。

测试路径遵循源码 1:1 镜像：

- `tests/unit/frontend/components/editor/PageSequenceEditor.test.tsx`
- `tests/unit/frontend/lib/page-sequence.test.ts`

### 开发环境集成测试

在 `GlobalConfigEditor`/`RadarEditor` 实际状态链路中验证一次完整排序，确保页面顺序、对比索引、当前页、时长统计和预览配置一致。测试不依赖进程外网络。

### 测试环境集成测试

当测试系统提供真实环境时，增加或扩展 Playwright 用户旅程，验证鼠标拖动绑定组、键盘排序、复制、删除确认和刷新后顺序持久化。该测试按用户旅程放在 `tests/testenv-integration/frontend/`，不镜像源码目录。

## 验收标准

- 所有页面以单列 bar 展示，操作区固定在右侧。
- 用户可通过鼠标、触控或键盘调整页面顺序。
- 对比组不能被拆分、不能内部换序，并在拖动任一成员时整体移动。
- 复制和删除绑定成员严格遵循已确认规则。
- 排序后页面、对比关系、当前页、时长和预览保持一致。
- 背景音乐为紧凑资源行，逐页时长默认折叠。
- 无效拖拽不改变配置；删除绑定成员必须确认。
- 前端 unit 与 dev-integration 测试全部通过并满足覆盖率门槛。
